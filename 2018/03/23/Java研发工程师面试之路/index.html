Java研发工程师面试之路

23 3月 2018

有点难受，最近在春招，投了两份简历，两次远程面试后都在等结果。

阿里

一面

一面了两次，都是结合简历和项目，第一次比较紧张，第二次就发挥的比较不错，第二次聊的很开心。主要是有了第一次的经验，也问了点java基础部分，还可以。
等了三天以为凉了。。

二面

周五晚上阿里第二轮面试，感觉有点凉。

自我介绍，然后是介绍自己的有点和长处，
我作死般说了个我好学。。。（要求举例）
中间问到了安全的一些东西、没细问
研发岗你的优势
之后就是问了些JAVA基础，回答的有点惨。

JAVA类的基类是什么？（Object类）
有哪些方法？（不会，记不得了）
判断两个对象相等怎么判（只记得好像是判断两个地址要相等，具体的不会）
问到equals和==的区别（扯半天没说出啥来）
多线程的实现方法（三种，继承Thread类、实现接口、线程池）{
    run和start方法的作用（反了，太久没碰过这些了，有点懵）
    线程池只是听说过
}

一个数学题，没计算说了个错答案，后来计算之后给了正确答案（有点紧张、当时很不确定）

还没出结果，怀着侥幸心理愿过。

更新： 3-27 官网上信息已更新为rejected 阿里算是无缘了。

java面试题：
java为什么是平台无关的？ （java虚拟机，.class字节码文件。半编译半解释）
.class文件和.java文件的区别
.class文件执行的过程（扯了半天没说对）
classpath
java异常
面向对象语言的特点（继承、多态、封装） 多态 （编译时多态：函数重载 运行时多态：继承和接口）

树的遍历
hash  hash冲突处理  （开放寻址法、再哈希法、链地址法、公共溢出区）
tcp三次握手过程   （为什么非得三次握手

momemta

一面

一面也是面了两个人。

第一个面试官是做C++研发的，聊了点简历上的项目，知道我C++工程方面没接触过之后放弃了给我的代码题。

第二个面试官是做视觉算法的，聊了点后，开始给我出算法题。

1.求两个矩形的交叉面积

（一下子联想到了矩形面积并）扩展开来讲了好久的实现原理，最后还扯了点线段树的原理（好像讲到了树状数组，有点。。）
其实答案后来面试后仔细考虑了挺简单的，就是先max、min判一下交叉，然后中间两个x的差乘中间两个y的差。。。

2.求两条平行线段的距离

1）平行于x的
2）斜着的
3）曲线的（弯道）
3）曲率不相同的

总结：

其实最后两道题都是迷迷糊糊的没给出实际的正解，大概是沟通的不到位，可能误解了面试官的意思。总的来说面试官真的很和善的，没有太过于紧张。之后就是等结果了。

就感觉momemta的面试官问的问题都给出了实际的应用场景，是属于需要解决的实际需求，然后自己也没给出实际的正解，第二道题满脑子想的全是骚骚的数学方法，醉醉的。后来自己都说不下去了。

最近等面试打乱了自己的复习计划，有点累，有点忐忑。
可能这些就是找工作的痛吧。
如果这两家没过，自己就像大面积投了，然后希望能有个好的结果吧。

    编程题：一道简单的贪心策略题，更重思路，一开始想法有缺陷，后来n方最后一秒过了（最后的心情……连问的问题都。。）
