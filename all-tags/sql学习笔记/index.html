sql学习笔记（待更新）

01 8月 2018

最近几天都在写数据处理的工具

包括阿里云 maxcompute原ODPS上的hive和本地mysql的纯sql语句，在实现功能的过程中收获颇多。

笔记是后来整理的，可能不全

mysql中varchar转float
float类型一直处于一个比较尴尬的地位，在这次的处理过程中更是深有体会。网上转类型几乎都是DECIMAL类型的，毕竟精度更高、更灵活，但关于float的很少见，及时找到也几乎都是错误的答案。
最后机缘巧合之下用round处理成功了，也是侥幸。

sql中变量的使用
因为我写的处理sql脚本是处理多个城市的，而处理的表因为涉及到所有的poi_info，表非常大，若不先进性一些处理会导致执行时间太长，影响正常使用，而因为处理的太多太杂，导致若复用代码会修改多处相同的细节，易出错，转换成变量之后就非常顺利，只需要修改一处就可以了。

关于sql中变量作为表名
变量存储的字符串，即传统意义上的String，若直接用变量替换表名用于from之后，dbma会不接受，报错。
经过搜索后学到了prepare、execute处理，即类似于java中jdbc的构造sql语句然后执行的过程。
其用法比较灵活，因为了解的不透彻，就不赘述
