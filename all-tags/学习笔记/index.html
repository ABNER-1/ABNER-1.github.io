设计模式学习笔记（待更新）

01 8月 2018

设计模式

单一职责原则 SRP

就一个类而言，应该仅有一个引起它变化的原因

软件设计真正要做的很多内容，就是发现职责并把这些职责相互分离

如果你想到多于一个动机去改变一个类，那这个类就具有多于一个的职责

开放-封闭原则 OCP

开放-封闭原则，是说软件实体（类、模块、函数等）应该可以扩展，但是不可修改

面对新的需求，对程序的改动是通过增加新代码进行的，而不是更改现有代码

  可维护，可扩展，可复用，灵活性好

依赖倒转原则

  1. 高层模块不应该依赖底层模块，两个都应该依赖抽象
  2. 抽象不应该依赖细节，细节应该依赖抽象

里氏代换原则

子类型必须能够替换掉它们的父类型

迪米特法则

如果两个类不必彼此直接通信，那么这两个类就不应当发生直接的相互作用。
如果其中一个类需要调用另一个类的某一个方法的话，可以通过第三者转发这个调用。

接口隔离原则

不应该依赖它不需要的接口；一个类对另一个类的依赖应该建立在最小的接口上。

常见设计模式

单例模式 Singleton

单例模式，保证一个类只有一个实例，并提供一个访问它的全局访问点。
  • 饿汉式单例类

    在自己被加载时就将自己实例化
  • 懒汉式单例类

    在第一次被引用时才会自己实例化

    多线程环境下，双重锁定保证单例且提高性能

适配器模式

将一个类的接口转换成客户希望的另外一个接口。
Adapter模式使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。

两种模式：类适配器模式、对象适配器模式

关于SSM框架的一些理解

05 7月 2018

SSM框架

最近一段时间都在接触SSM框架，之前可能对其理解不到位，有了很多错误的理解。这次实训也的的确确学到了不少。

SSM 由 Spring MVC + Spring boot + Mybatis三部分组成。

  • Spring MVC负责view层的显示
  • Spring boot负责业务模型
  • Mybatis负责与数据库的交互

值得一提的是maven，maven是搭建框架的底层，我的理解是为项目引入种种包，若不适用maven进行配置，可以参考写代码时手动引入依赖包的过程。具体实现方面不算是特别了解，应该和gradle差不多。

maven中最重要的是pom.xml文件，里面定义了整个项目的maven配置，其中包括版本、jdk版本、tomcat版本以及包依赖等等。

web项目中重要的配置文件是web.xml，其中定义了spring的配置、springMVC的配置以及前端到controller层的字符类型（若不进行配置，默认的编码会导致中文乱码）

通常web.xml中几个文件会单独写一个xml文件方便配置。
如 spring-mvc.xml文件中包含的是springmvc的配置，包含了controller层的扫描位置（方便url解析），以及controller层中跳转的string处理，例如加前缀后缀等。

applicationContext.xml包含了spring中各种配置，包含各层的扫描位置，数据库配置文件的引入，数据库处理类型的声明等。

更多的log4j等配置暂时没有了解的更详细。

关于在maven目录中引入的kaptcha验证码组件等、以及如何在ssm项目中引入开源项目库等还需要更多的练习去学习。

Java内存模型

23 3月 2018

JAVA内存模型

JMM为JVM一部分

并发编程 – 两个关键问题：线程之间的通信和同步

线程间通信 –共享内存 和 消息传递

共享对象：隐式通信 – 如共享对象

消息传递： 显示通信 – wait()和notify()

内存间交互操作

  • Lock：锁定，作用于主内存的变量，他把变量标识为一个线程独占的状态。
  • Unlock: 解锁，作用于主内存的变量，把一个变量的值从主内存读取到线程工作内存中，以便随后的load动作使用
  • read：读取，作用于主内存的变量，把一个处于锁定状态的变量释放出来，释放后的变量才可以被其他线程锁定
  • load：载入，作用于工作内存的变量，把read操作从主内存中得到的变量值放入工作内存的变量副本中
  • use：使用，作用于工作内存的变量，把工作内存中一个变量的值传递给执行引擎，每当虚拟机遇到一个需要使用的变量的值的字节码指令将会之星这个操作
  • assign：赋值，作用于工作内存的变量，把从执行引擎收到的值赋给工作内存的变量，每当虚拟机遇到一个给变量赋值的字节码指令时执行这个操作。
  • store：存储，作用于工作内存的变量，把工作内存中的一个变量的值赋给主内存中，以便随后的write操作使用
  • write：写入，作用于主内存的变量，把store操作从工作内存中得到的变量的值放入主内存的变量之中。

内存模型

  • 程序计数器

    每个线程有要有一个独立的程序计数器，记录下一条要运行的指令。线程私有的内存区域。如果执行的是JAVA方法，计数器记录正在执行的java字节码地址，如果执行的是native方法，则计数器为空。

  • 虚拟机栈

    线程私有的，与线程在同一时间创建。管理JAVA方法执行的内存模型。

  • 本地方法区

    和虚拟机栈功能相似，但管理的不是JAVA方法，是本地方法

  • 方法区

    线程共享的，用于存放被虚拟机加载的类的元数据信息：如常量、静态变量、即时编译器编译后的代码。也称为永久代。

  • JAVA 堆

    线程共享的，存放所有对象实例和数组。垃圾回收的主要区域。可以分为新生代和老年代(tenured)。
