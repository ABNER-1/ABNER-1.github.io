更全面的解释

PS: CyclicBarrier的用法和 区别】

CyclicBarrier初始化时规定一个数目，然后计算调用了CyclicBarrier.await()进入等待的线程数。当线程数达到了这个数目时，所有进入等待状态的线程被唤醒并继续。

CyclicBarrier初始时还可带一个Runnable的参数， 此Runnable任务在CyclicBarrier的数目达到后，所有其它线程被唤醒前被执行。

java 接口中定义变量的意义

01 8月 2018

JAVA interface中定义变量

接口定义了行为的协议，而不是行为如何执行实现。实现接口的类支持该接口中定义的行为协议。

个人理解
接口中定义的变量可以视为是要被实现类所认可的协议，
其默认用public static final修饰，具体解释如下。
所有此变量和在所有实现类中定义的变量于功能和性质上是有区别的。

接口中声明的所有字段都是public static final的。为什么？
如果一个变量没有被定义为final，任何类的实现都可以改变变量的值。同时他就会变成类的实现的一部分，而接口是一个不带任何实现的<b>纯粹的规范</b>；
如果变量是静态的，那么这个变量就是属于接口的，而不是属于实例对象或者运行时的对象的。（注：由于接口不能被实例化，所以，定义为非静态，其实也没有意义。）；
接口定义了调用者如何跟接口的实现类的实例对象交互，所以如果成员不是public，那么调用者没办法去访问它；
接口的每一个字段的声明都必须是pubic static final 的，它允许我们在声明时指定所有或者部分修饰符。同时，任何字段的声明必须有一个初始化表达式，它可以不用一定是一个常量表达式，它的计算和赋值只会进行一次，然后这个接口字段就被初始化了

PS：接口中不可使用static代码块。

01 8月 2018

句点符号

通配符，匹配所有符号

方括号符号

[],匹配方括号里面指定的字符才参与匹配，只匹配单字符

或符号 |

解决[]只能匹配一个字符的问题，可匹配多个 单字符或多字符

代表匹配次数的符号

  • * 0或多次

  • + 1或多次

  • ? 0或1次

  • {n} 恰好n次

  • {n,m} 从n到m次都可以

- 和 \- 区别
-表示一个范围 。如[0-9]代表匹配0-9的数字
\-则是代表匹配”-“字符本身
即用转义符\处理。

否符号^

[^x]匹配不是x的字符

圆括号与空白符号

常用符号

  • \s [\t\n\r\f]

java异常 在分层设计中的理解

01 8月 2018

java异常

多层之间的异常，每一层只要关注下一层的异常，
若此层无法解决，则封装成这一层的异常抛给上一层。

如manage层 调用dao层的方法，此时manage因为知道调用的下一层是dao层，其常见的异常有哪些，此时进行try尝试处理，如果解决则不上抛，若解决不了则应包装成manage层的异常进行上抛。（此时manage层的异常不一定和dao层抛出的异常一致，就好像有时在try-catch时，直接e.printStackTrace()一样只是在命令行打印异常信息在程序中出错的位置及原因，但并未进行解决）

throws和throw关键字

throws就是放在类与方法后面,然后你要调用这个类与方法时,必须try-catch对应的异常

throw就是单个语句抛出异常
