Java内存模型

3月 23, 2018

JAVA内存模型

JMM为JVM一部分

并发编程 – 两个关键问题：线程之间的通信和同步

线程间通信 –共享内存 和 消息传递

共享对象：隐式通信 – 如共享对象

消息传递： 显示通信 – wait()和notify()

内存间交互操作

  • Lock：锁定，作用于主内存的变量，他把变量标识为一个线程独占的状态。
  • Unlock: 解锁，作用于主内存的变量，把一个变量的值从主内存读取到线程工作内存中，以便随后的load动作使用
  • read：读取，作用于主内存的变量，把一个处于锁定状态的变量释放出来，释放后的变量才可以被其他线程锁定
  • load：载入，作用于工作内存的变量，把read操作从主内存中得到的变量值放入工作内存的变量副本中
  • use：使用，作用于工作内存的变量，把工作内存中一个变量的值传递给执行引擎，每当虚拟机遇到一个需要使用的变量的值的字节码指令将会之星这个操作
  • assign：赋值，作用于工作内存的变量，把从执行引擎收到的值赋给工作内存的变量，每当虚拟机遇到一个给变量赋值的字节码指令时执行这个操作。
  • store：存储，作用于工作内存的变量，把工作内存中的一个变量的值赋给主内存中，以便随后的write操作使用
  • write：写入，作用于主内存的变量，把store操作从工作内存中得到的变量的值放入主内存的变量之中。

内存模型

  • 程序计数器

    每个线程有要有一个独立的程序计数器，记录下一条要运行的指令。线程私有的内存区域。如果执行的是JAVA方法，计数器记录正在执行的java字节码地址，如果执行的是native方法，则计数器为空。

  • 虚拟机栈

    线程私有的，与线程在同一时间创建。管理JAVA方法执行的内存模型。

  • 本地方法区

    和虚拟机栈功能相似，但管理的不是JAVA方法，是本地方法

  • 方法区

    线程共享的，用于存放被虚拟机加载的类的元数据信息：如常量、静态变量、即时编译器编译后的代码。也称为永久代。

  • JAVA 堆

    线程共享的，存放所有对象实例和数组。垃圾回收的主要区域。可以分为新生代和老年代(tenured)。

计网复习摘要

1月 12, 2018

这是考试前的复习笔记，现在草草整理一下，有空详细整理。

计算机网络

计算机网络定义：已能够相互共享资源的方式互联起来的自治计算机系统的集合。
特征：共享软件、硬件以及数据等资源；分布在不同地理位置的“自治系统”；遵循一定的规则——协议

计算机网络和分布式系统区别：计算机网络目的是共享资源，对外呈现分散，分布式系统目的是以提高整体性能，对外呈现整体。

分层的优点：各层独立、灵活性好、结构上分开、促进标准化

数据链路层

数据链路层协议提供的最基本的功能：数据在数据链路上的正常传输、帧定界（帧同步）、差错控制、流量控制、寻址

向网络层提供的三种基本服务：无确认无连接服务、有确认无连接服务、面向连接确认服务

应答方式：正向应答、负向应答、双向应答

成帧：字符计数法、带字符填充的首尾字符定界法、带位填充的首尾标记定界法（+01111110，五个1后加0）、物理层编码违例法

差错控制
随机、连续突发
帧出错、帧丢失
方法：自动纠错机制（FEC）,检错反馈重发机制（ARQ）
码字、纠错码、检错码

编码的海明距离
海明定理
检测d位错误，编码海明距离至少为d+1
纠正 2d+1

差错码举例：奇偶校验码、垂直水平奇偶校验码、交织奇偶校验码
循环冗余码CRC
检验和算法

单工停等协议
连续ARQ协议
滑动窗口协议：控制已发送但未确认的数据帧的个数（滑动窗口的大小）
接受窗口

SLIP协议
PPP协议（包括链路控制协议LCP、网络控制协议NCP）

局域网

局域网是一种将小区域内的各种通信设备互联在一起的通信网络

IEEE 802局域网：数据链路层（逻辑链路控制子层、介质访问控制子层）、物理层

物理层功能：信号编码\译码、前导码的生成\去除、比特的发送\接收

数据链路层的功能：

MAC子层：成帧\拆帧，实现、维护MAC协议，位差错检测，寻址

LLC子层：向高层提供SAP，建立/释放逻辑连接，差错控制，帧序号处理，提供某些网络层功能

信道共享技术
关键问题：如何解决对信道争用

解决信道争用的协议：介质访问控制协议MAC，是数据链路层协议的一部分

静态分配：频分多路复用FDM（波分复用WDM）、时分多路复用TDM
动态分配：冲突协议、无冲突协议
