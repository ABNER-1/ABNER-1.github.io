题解-乘积

牛客网-乘积题解

题目链接

题面简要如下：

  小易给定你一个长度为n的正整数序列Ai，你每次可以使用1的代价将某个数加一或者减一，你希望用最少的代价使得所有数的乘积等于B，求最小代价（操作结束后每个数也必须是正整数）。

简要分析：

可以很容易的定义出转移方程：

f[i][j]: 前i个数乘积为j的最小代价
b[i]： 可能的a[i]最后的值，范围为[1,B]

f[i][j] = min(f[i][j] , f[i][j/b[i]] + abs(b[i] - a[i]))
当且仅当 j/b[i] 为整数时有效

可以很简单的得出时间复杂度O(n * B * B)，大概是10^13的级别，很显然是不可行的。

简单分析：

  • 当B确定的时候，所有可能的j的值一定是B的因子，那么b[i]的取值就缩小到log(B)级别。
  • 那么j/b[i]有效的值 也必然保证了 b[i] 是 j 的因子，其也是 B 的因子的子集。那么对 b[i] 的遍历也一定是log(B)的级别。

此时的时间复杂度是O(n * log(B) * log(B))，也即是O(n * B)大概是10^8级别，在可通过的范围内。

当然，这是一个很经典的背包模型，可以对第一维进行优化，将O(n * B)的级别优化到O(B),甚至可以根据这道题离散化一下的特性，可以优化到O(log(B)),当然这就没太大必要了。

附代码

#include<iostream>
#include<algorithm>
#include<vector>
#include<cmath>

using namespace std;
const int maxn = 1e5 + 5;
int f[2][maxn];

int main(){
    int n, B;
    vector<int> a;
    vector<int> fac_b;

    cin >> n >> B;
    a.resize(n);
    for(int i = 0; i < n; ++i ){
        cin >> a[i];
    }

    // calc all factors of B
    for(int i = 1; i <= B; ++i ) {
        if(B % i == 0) {
            fac_b.push_back(i);
        }
    }
    // init f[0][j]
    for(auto& item : fac_b) {
        f[0][item] = abs(item - a[0]);
    }

    for (int i =1; i < n; ++i ) {
        for( auto& j : fac_b ) {
            // a[i] -> 1
            f[i & 1][j] = f[(i -1) & 1][j] + abs(a[i] - 1);
            // a[i] -> k
            for( auto& k : fac_b ) {
                if( j < k ) break;
                if( j % k > 0 ) continue;
                f[i & 1][j] = min(f[i&1][j] , f[(i-1) & 1][j / k] + abs(k - a[i]));
            }
        }
    }
    cout << f[(n-1)&1][B] << endl;

    return 0;
}

第二种做法

思路

  1. 求出B的所有因子
  2. 遍历出所有乘积为B， 长度为n的因子序列
  3. 算出每个可能的因子序列 与 原数组a的代价
